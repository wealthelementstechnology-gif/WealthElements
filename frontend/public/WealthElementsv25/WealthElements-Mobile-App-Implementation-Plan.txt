# WealthElements Mobile App - Implementation Plan

## Overview

Building a React Native mobile application for comprehensive financial planning that provides:

- 8 Life Events Goal Planner (Education, Marriage, Home, Vehicle, Vacation, Retirement, Emergency, Other)
- Financial Snapshot (Net Worth & Cash Flow Analysis)
- 11 Financial Calculators
- Mutual Fund Analyzer with real-time data from mfapi.in
- Income Tax Calculator (Old vs New Regime)
- ML-based Budget Optimization
- Goal tracking and progress visualization

**Repositories:**

- **wealthelements-app** - React Native mobile app (new, to be created)
- **wealthelements-backend** - Node.js backend API (new, to be created)

**Design Style:** Minimal Clean aesthetic (matching current web design system)

- Light background (#F0F3FA light mode, dark mode support)
- Clean typography with clear hierarchy (Inter font family)
- Generous spacing and padding
- Simple icons and minimal borders
- Green primary accent (#2E7D32 for positive/growth)
- Focus on content over decoration

## Current State

Vanilla JavaScript web application with:

- HTML5, CSS3, Vanilla JavaScript
- Chart.js for visualizations
- React (limited use - only in Mutual Fund Analyzer module)
- TensorFlow.js for ML optimization (experimental)
- localStorage for data persistence
- mfapi.in API integration for mutual fund data
- No backend server (fully client-side)
- No authentication system

## Technology Decisions

### Core Framework

- React Native (CLI, not Expo) with TypeScript
- Navigation: React Navigation 6.x
- State Management: Redux Toolkit with RTK Query
- API Client: Axios for HTTP requests
- Authentication: Biometric (Touch ID/Face ID) + 6-digit PIN + optional phone OTP

### Visualization Libraries

- **victory-native** - Primary charting library (powerful, customizable)
- react-native-svg - SVG support (required for victory-native)
- react-native-chart-kit - Secondary option for simple charts

### Required Libraries

- @react-navigation/native - Navigation
- @react-navigation/bottom-tabs - Bottom tab navigation
- @react-navigation/stack - Stack navigation for modals
- @reduxjs/toolkit - State management
- react-redux - Redux bindings
- axios - API client for backend communication
- date-fns - Date manipulation and formatting
- react-native-svg - SVG support (required for victory-native)
- victory-native - Chart visualizations
- react-native-encrypted-storage - Secure local storage for financial data
- react-native-biometrics - Biometric authentication
- react-native-keychain - Secure credential storage
- @tensorflow/tfjs-react-native - ML model integration
- react-native-linear-gradient - UI gradients
- react-native-reanimated - Smooth animations
- Additional libraries TBD

### Data Storage Strategy

- **Local Storage:** react-native-encrypted-storage for sensitive financial data
- **Offline-First:** SQLite database for offline functionality
- **Security:** All financial data, goal details, and calculations encrypted at rest
- **Sync Strategy:** Local-first with cloud backup when online

### Phase 2: Backend Implementation

#### Repo: wealthelements-backend

**Section 1: Authentication System**

**File: src/models/User.model.ts**
**Purpose:** User data model
**Fields:**

- userId (UUID, primary key)
- phoneNumber (string, optional, for OTP auth)
- pinHash (string, bcrypt hashed 6-digit PIN)
- biometricEnabled (boolean, default false)
- biometricPublicKey (string, optional)
- fullName (string)
- age (number)
- city (string, optional)
- maritalStatus (enum: SINGLE, MARRIED, COUPLE_MODE)
- partnerName (string, optional, for couple mode)
- partnerAge (number, optional)
- createdAt (timestamp)
- updatedAt (timestamp)
- lastLoginAt (timestamp)
- onboardingCompleted (boolean, default false)

**File: src/controllers/auth.controller.ts**
**Purpose:** Handle authentication requests
**Endpoints:**

**POST /api/auth/register**

- Request body: { pin: string (6 digits), fullName: string, phoneNumber?: string }
- Validation: PIN must be exactly 6 numeric digits
- Process: Hash PIN with bcrypt (10 rounds), create user record, generate JWT token
- Response: { userId, token, expiresIn }
- Error handling:
  - 400 if PIN format invalid
  - 500 for database errors

**POST /api/auth/login**

- Request body: { userId: string, pin: string }
- Process: Verify PIN hash, update lastLoginAt, generate new JWT token
- Response: { token, expiresIn, user: { userId, fullName, biometricEnabled } }
- Error handling:
  - 401 if user not found or PIN incorrect
  - 500 for database errors

**POST /api/auth/enable-biometric**

- Request headers: Authorization: Bearer <token>
- Request body: { biometricPublicKey: string }
- Process: Store biometric public key, set biometricEnabled = true
- Response: { success: true }

**POST /api/auth/verify-biometric**

- Request body: { userId: string, biometricSignature: string }
- Process: Verify biometric signature against stored public key, generate JWT token
- Response: { token, expiresIn }

**Section 3: 8 Life Events Goals API**

**File: src/models/Goal.model.ts**
**Purpose:** Store life event goals
**Fields:**

- goalId (UUID, primary key)
- userId (UUID, foreign key)
- name (string)
- type (enum: EDUCATION, MARRIAGE, HOME, VEHICLE, VACATION, RETIREMENT, EMERGENCY, OTHER)
- targetAmount (decimal, today's value)
- futureValue (decimal, inflation-adjusted)
- inflationRate (decimal, based on goal type)
- targetDate (timestamp)
- yearsLeft (number, calculated)
- priority (enum: HIGH, MEDIUM, LOW)
- requiredSIP (decimal, calculated)
- currentValue (decimal, current savings towards goal)
- expectedReturn (decimal, assumed rate of return)
- isRetirementGoal (boolean, special calculation)
- retirementMonthlyExpense (decimal, for retirement only)
- retirementLifeExpectancy (number, for retirement only)
- isCompleted (boolean)
- status (enum: ACTIVE, ON_TRACK, BEHIND, PAUSED, COMPLETED)
- mlOptimized (boolean, if ML adjusted values)
- originalTargetAmount (decimal, before ML optimization)
- createdAt (timestamp)
- updatedAt (timestamp)

**File: src/services/goals.service.ts**
**Purpose:** Goal calculation and optimization logic

**Method: calculateGoalFutureValue()**

- Parameters: targetAmount, yearsLeft, inflationRate
- Formula: FV = PV * (1 + r)^n
- Returns: futureValue (decimal)

**Method: calculateRequiredSIP()**

- Parameters: futureValue, yearsLeft, expectedReturn
- Formula: SIP = FV * r / ((1+r)^n - 1) where r is monthly rate
- Returns: requiredSIP (decimal)

**Method: calculateRetirementCorpus()**

- Parameters: monthlyExpense, retirementAge, lifeExpectancy, inflationRate, postRetirementReturn
- Process:
  1. Calculate years in retirement = lifeExpectancy - retirementAge
  2. Calculate inflation-adjusted expense at retirement
  3. Calculate required corpus using annuity formula
- Returns: { corpus, inflationAdjustedMonthlyExpense, yearsInRetirement }

**Method: optimizeGoalsBudget()**

- Parameters: goals[], availableBudget, userProfile
- Process:
  1. Calculate total SIP required for all goals
  2. If total > availableBudget:
     - Apply ML-based optimization OR
     - Apply rule-based constraints (reduce by priority, extend timeline)
  3. Return optimized goals with adjusted SIPs
- Returns: { optimizedGoals[], shortfall, recommendations[] }

**Section 4: Financial Calculators API**

**File: src/services/calculator.service.ts**
**Purpose:** All financial calculation formulas

**Method: futureValueLumpsum()**

- Parameters: principal, rate, years
- Formula: FV = P * (1 + r)^n
- Returns: futureValue

**Method: lumpsumForFutureValue()**

- Parameters: futureValue, rate, years
- Formula: P = FV / (1 + r)^n
- Returns: requiredLumpsum

**Method: sipFutureValue()**

- Parameters: monthlyAmount, rate, years
- Formula: FV = PMT * (((1+r)^n - 1) / r) * (1+r)
- Returns: futureValue

**Method: sipForFutureValue()**

- Parameters: futureValue, rate, years
- Formula: PMT = FV * r / (((1+r)^n - 1) * (1+r))
- Returns: requiredSIP

**Method: swpCalculator()**

- Parameters: corpus, monthlyWithdrawal, rate, inflationRate
- Process: Calculate how long corpus will last with inflation-adjusted withdrawals
- Returns: { yearsLasting, totalWithdrawn, finalBalance }

**Method: lumpsumForSWP()**

- Parameters: monthlyWithdrawal, rate, years, inflationRate
- Returns: requiredCorpus

**Method: limitedPeriodSIPFutureValue()**

- Parameters: monthlyAmount, sipYears, rate, totalYears
- Process: Calculate FV after SIP stops and grows for remaining period
- Returns: futureValue

**Method: limitedPeriodSIPForGoal()**

- Parameters: futureValue, sipYears, rate, totalYears
- Returns: requiredSIP

**Method: comboFutureValue()**

- Parameters: lumpsum, monthlyAmount, rate, years
- Process: Calculate combined FV of lumpsum + SIP
- Returns: { lumpsumFV, sipFV, totalFV }

**Method: lumpsumRequiredWithSIP()**

- Parameters: futureValue, monthlyAmount, rate, years
- Returns: requiredLumpsum (given known SIP)

**Method: sipRequiredWithLumpsum()**

- Parameters: futureValue, lumpsum, rate, years
- Returns: requiredSIP (given known lumpsum)

**Method: stepUpSIPFutureValue()**

- Parameters: initialSIP, annualStepUp, rate, years
- Process: Calculate FV with yearly SIP increase
- Returns: { futureValue, totalInvested, breakdown[] }

**Purpose:** Configure Redux store

```typescript
import { configureStore } from '@reduxjs/toolkit';
import authReducer from './slices/authSlice';
import userReducer from './slices/userSlice';
import goalsReducer from './slices/goalsSlice';
import snapshotReducer from './slices/snapshotSlice';
import calculatorReducer from './slices/calculatorSlice';
import mutualfundReducer from './slices/mutualfundSlice';
import taxReducer from './slices/taxSlice';
import { apiSlice } from './api/apiSlice';

export const store = configureStore({
  reducer: {
    auth: authReducer,
    user: userReducer,
    goals: goalsReducer,
    snapshot: snapshotReducer,
    calculator: calculatorReducer,
    mutualfunds: mutualfundReducer,
    tax: taxReducer,
    [apiSlice.reducerPath]: apiSlice.reducer,
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware().concat(apiSlice.middleware),
```

```typescript
export const CalculationService = {
  // Future Value of Lumpsum
  futureValueLumpsum(principal: number, rate: number, years: number): number {
    const r = rate / 100;
    return principal * Math.pow(1 + r, years);
  },

  // Future Value of SIP
  futureValueSIP(monthlyAmount: number, annualRate: number, years: number): number {
    const r = annualRate / 100 / 12;
    const n = years * 12;
    return monthlyAmount * (((Math.pow(1 + r, n) - 1) / r) * (1 + r));
  },

  // Required SIP for Goal
  requiredSIP(futureValue: number, annualRate: number, years: number): number {
    const r = annualRate / 100 / 12;
    const n = years * 12;
    return (futureValue * r) / (((Math.pow(1 + r, n) - 1)) * (1 + r));
  },

  // Inflation Adjusted Future Value
  inflationAdjusted(amount: number, inflationRate: number, years: number): number {
    const r = inflationRate / 100;
    return amount * Math.pow(1 + r, years);
  },

  // Retirement Corpus Calculation
  retirementCorpus(
    monthlyExpense: number,
    yearsToRetirement: number,
    yearsInRetirement: number,
    inflationRate: number,
    postRetirementReturn: number
  ): number {
    // Inflation-adjusted expense at retirement
    const retirementExpense = this.inflationAdjusted(monthlyExpense, inflationRate, yearsToRetirement);

    // Required corpus (present value of annuity)
    const r = postRetirementReturn / 100 / 12;
    const n = yearsInRetirement * 12;
    return retirementExpense * ((1 - Math.pow(1 + r, -n)) / r);
  },
};
```

```typescript
export const formatCurrency = (amount: number): string => {
  // Format as Indian currency
  // 2450000 → Rs. 24,50,000
  const formatter = new Intl.NumberFormat('en-IN', {
    style: 'currency',
    currency: 'INR',
    minimumFractionDigits: 0,
    maximumFractionDigits: 0,
  });
  return formatter.format(amount).replace('₹', 'Rs. ');
};

export const formatCompactCurrency = (amount: number): string => {
  // Format in compact form
  // 2450000 → Rs. 24.5L
  // 150000 → Rs. 1.5L
  // 15000 → Rs. 15K
  if (amount >= 10000000) {
    return `Rs. ${(amount / 10000000).toFixed(1)}Cr`;
  } else if (amount >= 100000) {
    return `Rs. ${(amount / 100000).toFixed(1)}L`;
  } else if (amount >= 1000) {
    return `Rs. ${(amount / 1000).toFixed(1)}K`;
  }
  return formatCurrency(amount);
};

export const formatPercentage = (value: number): string => {
  return `${value.toFixed(2)}%`;
};
```

**WealthElements Mobile App** is a comprehensive financial planning application that converts the existing web application into a native mobile experience. It includes:

1. **8 Life Events Planner** - Goal-based financial planning for major life milestones
2. **Financial Snapshot** - Net worth and cash flow tracking
3. **11 Financial Calculators** - Various investment and goal planning tools
4. **Mutual Fund Analyzer** - Real-time fund data with 47+ metrics
5. **Tax Calculator** - Old vs New regime comparison
6. **ML Optimization** - Smart budget allocation when goals exceed budget

**Key Differentiators:**

- Offline-first architecture with cloud sync
- Biometric authentication for security
- Category-specific inflation rates for accurate projections
- ML-based budget optimization
- Real-time mutual fund data integration
- Comprehensive tax planning tools

**Tech Stack:**

- React Native (CLI) with TypeScript
- Node.js + Express backend
- MongoDB for data storage
- Redis for caching
- TensorFlow.js for ML models
- mfapi.in for mutual fund data

This implementation plan provides a complete roadmap for converting WealthElementsv25 into a production-ready mobile application while preserving all existing functionality and adding mobile-specific enhancements.
