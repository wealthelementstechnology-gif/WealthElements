PROBLEM IDENTIFIED:
══════════════════════════════════════════════════════════════════════

The feedback dialog was not appearing after clicking OPTIMIZE.

ROOT CAUSE:
──────────────────────────────────────────────────────────────────────

In step6.js line 1517, the OPTIMIZE button was set up like this:

  optimizeBtn.addEventListener('click', optimizePlan);
                                        ^^^^^^^^^^^
                                        Direct reference!

This created a DIRECT reference to the original optimizePlan function.

When step6-ml-integration.js tried to wrap the function:

  window.optimizePlan = function() {
    // ML tracking wrapper
    originalOptimizePlanFunction.call(this);
  };

The button STILL had the direct reference to the OLD function, so it
was bypassing the wrapper entirely!

Think of it like this:

  OLD CODE:
  ─────────
  1. optimizePlan function created
  2. Button points directly to optimizePlan ←─┐
  3. window.optimizePlan = optimizePlan       │ Direct reference
  4. ML wrapper replaces window.optimizePlan  │ stored here!
  5. Button STILL points to original ─────────┘ (bypasses wrapper!)


  NEW CODE:
  ─────────
  1. optimizePlan function created
  2. window.optimizePlan = optimizePlan
  3. ML wrapper replaces window.optimizePlan
  4. Button calls window.optimizePlan() ←──────┐
  5. Goes through wrapper! ────────────────────┘ (ML tracking works!)


══════════════════════════════════════════════════════════════════════

FIX APPLIED:
══════════════════════════════════════════════════════════════════════

Modified: step6.js (lines 1514-1527)

BEFORE:
───────
  // Add event listener for optimize button
  const optimizeBtn = document.getElementById('optimizeBtn');
  if (optimizeBtn) {
    optimizeBtn.addEventListener('click', optimizePlan);  // ❌ Direct reference
  }

  // Expose optimizePlan to window for ML integration
  window.optimizePlan = optimizePlan;


AFTER:
──────
  // Expose optimizePlan to window for ML integration (MUST be before addEventListener!)
  window.optimizePlan = optimizePlan;

  // Add event listener for optimize button
  // Use arrow function to call window.optimizePlan (allows ML wrapper to intercept)
  const optimizeBtn = document.getElementById('optimizeBtn');
  if (optimizeBtn) {
    optimizeBtn.addEventListener('click', function() {
      // Call through window.optimizePlan so ML integration can wrap it
      if (typeof window.optimizePlan === 'function') {
        window.optimizePlan();  // ✅ Calls through window (ML wrapper intercepts!)
      }
    });
  }


KEY CHANGES:
────────────

1. Moved window.optimizePlan = optimizePlan BEFORE the button setup
2. Changed addEventListener to call window.optimizePlan() instead of direct reference
3. Added safety check: if (typeof window.optimizePlan === 'function')
